This document provides technical details on the 4 deployment architectures available for Mage Pro. This will help technical teams identify which deployment option best suits their current use case, data governance policies, and operational constraints. 

### Deployment Architecture Overview

Mage Pro's platform architecture consists of two primary components:

1. **Control Plane**: Handles orchestration, scheduling, monitoring, and metadata management
2. **Data Plane**: Executes pipeline code, processes data, and manages compute resources

These components can be deployed in various configurations depending on your technical and compliance requirements.
