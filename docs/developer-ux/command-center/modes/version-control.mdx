With the Version Control mode in the Command Center, you can initialize as many Git repositories
in a single Mage application as your desire.

The Version Control mode enables you to invoke Git commands for any Git repository without 
having to constantly change working directories.

## Activate mode

<AccordionGroup>
    <Accordion title="Transform into a version control master" icon="code-branch" iconType="duotone" color="#FFCC19" defaultOpen>
        <Frame>
            <p align="center">
            <img
                alt="Transform into a version control master"
                src="https://github.com/mage-ai/assets/raw/main/dev-ux/command-center/modes/version-control/activate.gif?raw=true"
            />
            </p>
        </Frame>
    </Accordion>
</AccordionGroup>

<Steps>
  <Step title="Launch Command Center">
    Open the Command Center. 
    For all the ways to do this, [read more here](/developer-ux/command-center/spellcaster#launch-command-center).
  </Step>
  <Step title="Search for mode">
    In the search bar, start typing the words `Version Control`.
  </Step>
  <Step title="Invoke action">
    Once you see the row titled <b>Activate version control mode</b>, 
    select that row and invoke its action.
  </Step>
</Steps>

        <Frame>

<Steps>
  <Step title="New Git project">
    Select the row title <b>New Git project</b>.
  </Step>
  <Step title="Select directory">
    Choose a location to initialize a Git repository. 
    The `git init` command will be executed in this directory and it’ll contain the `.git` folder.
  </Step>
  <Step title="Initialize repo">
    Click the button labeled <b>Initalize repo</b>.
  </Step>
</Steps>

## Other git actions

If a git action isn’t available in the Command Center, 
you can type in the search bar the full git command you want to run and click <kbd>`⌅ Enter`</kbd>
to submit the command.
